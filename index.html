Step A — Set up your Cloudflare proxy (one time only)
Square's API blocks direct browser requests for security. A free Cloudflare Worker acts as a middleman. This takes about 5 minutes and you never need to touch it again.
1

Go to workers.cloudflare.com and create a free account (or sign in).

2

Click Create application → Create Worker. Give it any name like bakery-proxy. Click Deploy.

3

Click Edit code. Select all the default code and delete it. Paste in the contents of the cloudflare-worker.js file you downloaded. Click Deploy.

4

Copy the Worker URL shown at the top (looks like https://bakery-proxy.yourname.workers.dev). Paste it into the Cloudflare Worker URL field above.

Step B — Get your Square credentials
1

Go to developer.squareup.com and sign in with your Square account.

2

Click + to create a new application — name it “Bakery Planner”. Click Save.

3

Open your app → click the Production tab → copy the Production Access Token and paste it above.

4

For location IDs: in your Square Dashboard, go to Account & Settings → Locations. Each location shows an ID starting with L.

5

Hit Test connection — if it works, move to the Catalog tab to sync your menu items.

6

To share with teammates: once you’ve set up your relationships, use Export settings (top right) to save a config file. Teammates open this file and hit Import settings — they only need to add their own Square token.

Ingredient relationships
For each menu item that uses a production item, enter how much it uses. Example: “Jambon Beurre” uses 0.5 of “Baguette”. The tool sums all relevant menu sales and converts to total production units needed.

Out-of-stock events
If an item sold out mid-day last week, log it here by day and location. The tool fetches that item’s actual hourly sales curve from Square for that day and uses it to estimate missed sales — so if baguettes sell mostly in the morning, it won’t over-credit afternoon hours.
How the hourly curve works: The tool looks at that specific item’s actual sales by hour from Square data. It calculates what fraction of the day’s sales typically happen during the hours after the item sold out, then applies that fraction to estimate missed units. If baguettes sell 80% before noon and sold out at 10am, only the remaining pre-noon hour gets credited — not a full afternoon.
